# Security Policy

RootLabs POS for WooCommerce handles operational data such as sales, cash sessions, refunds, employees, customers, payment records, audit logs, and optional Telegram notifications.

Please do not report security vulnerabilities in public GitHub issues.

## Reporting a vulnerability

Send a private report to:

hola@rootlabs.mx

Include:

- A clear description of the issue.
- Steps to reproduce.
- Affected version or commit.
- Relevant environment details: WordPress version, WooCommerce version, PHP version, and whether HPOS is enabled.
- Logs or screenshots only after sensitive information has been removed.

## Do not include

Please do not share:

- Telegram bot tokens.
- API keys.
- Passwords.
- Customer personal data.
- Production order data.
- Cash totals from real stores.
- Server paths.
- Private IP addresses.
- Cookies.
- Authorization headers.

## Supported versions

This repository is currently in an early public release stage. Security fixes will target the latest public release unless otherwise stated.
